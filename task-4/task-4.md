# Практическое задание №4

**О чём задание:** техническое проектирование. Необходимо нарисовать C4-диаграммы уровней Context и Containers, а также выбрать и обосновать стек технологий для MVP "Зелёного агента".

## C4-диаграммы

### Уровень 1: Контекст (Context)
На этом уровне система "Зелёный агент" представлена как единое целое (чёрный ящик). Показаны все внешние пользователи и интеграции.

**Основные элементы и связи:**

- **Пользователи:**
  - Клиент (создаёт заказ, отслеживает статус)
  - Курьер / Волонтёр (принимает заказ, обновляет статус доставки)
  - Администратор (модерирует пользователей, решает конфликты)

- **Внешние системы:**
  - Платёжный шлюз (YooKassa) - для списания средств с клиентов и выплат курьерам
  - Картографический сервис (Яндекс.Карты) - геокодирование адресов и построение маршрутов
  - Сервис push-уведомлений (Firebase Cloud Messaging) - оповещение о новых заказах и смене статуса
  - Почтовый сервис (SendGrid) - отправка чеков и отчётов

- **Связи:**
  - Все пользователи взаимодействуют с системой через мобильное приложение или веб-панель
  - Система отправляет запросы к внешним сервисам по API

### Уровень 2: Контейнеры (Containers)
Система разбивается на независимо разворачиваемые технические контейнеры: приложения, базы данных, хранилища, фоновые обработчики.

**Контейнеры:**

- **Мобильное приложение (Клиент)** - интерфейс для заказа вывоза вторсырья, отслеживания статуса и управления профилем.
- **Мобильное приложение (Курьер/Волонтёр)** - интерфейс для приёма заказов, навигации и обновления статуса.
- **Веб-панель администратора** - управление пользователями, мониторинг заказов, разрешение конфликтов.
- **API Gateway (Nginx)** - единая точка входа, маршрутизация запросов, обеспечение HTTPS, отдача статики.
- **Сервер приложений (Python + FastAPI)** - бизнес-логика, аутентификация, работа с заказами и баллами.
- **Фоновый обработчик (Celery Worker)** - асинхронные задачи: отправка уведомлений, формирование отчётов, взаимодействие с внешними API.
- **База данных (PostgreSQL + PostGIS)** - хранение данных о пользователях, заказах, гео-информации.
- **Кэш и брокер (Redis)** - кэширование и очереди задач для Celery.
- **Файловое хранилище (S3)** - хранение изображений (аватары, фото вторсырья).

**Внешние сервисы (отдельно):**
- YooKassa (платёжный шлюз)
- Яндекс.Карты API
- Firebase Cloud Messaging
- SendGrid (почтовый сервис)

**Связи:**
- Мобильные приложения и веб-панель обращаются к API Gateway
- API Gateway проксирует запросы на сервер приложений
- Сервер приложений взаимодействует с БД, Redis, файловым хранилищем и внешними сервисами
- Фоновый обработчик получает задачи из Redis и общается с push-сервисом и почтовым сервисом

## Стек технологий

| Технология | Назначение | Обоснование для MVP |
|------------|------------|----------------------|
| **Flutter** | Мобильные приложения (iOS/Android) | Один код для двух платформ, быстрая разработка, хорошая производительность, большое сообщество. |
| **Python + FastAPI** | Серверная часть (API) | Простой и быстрый фреймворк, встроенная асинхронность, автоматическая документация (Swagger), популярен в стартапах. |
| **PostgreSQL + PostGIS** | База данных | Надёжная реляционная СУБД, поддержка гео-расширений для поиска ближайших курьеров/заказов. |
| **Redis** | Кэш и брокер сообщений | Высокая скорость, идеален для кэширования и очередей задач, легко интегрируется с Celery. |
| **Celery** | Обработка фоновых задач | Стандартное решение для Python, позволяет вынести отправку уведомлений и другие длительные операции из основного потока. |
| **Nginx** | API Gateway, отдача статики | Проверенное решение, высокая производительность, простота настройки HTTPS и проксирования. |
| **YooKassa** | Приём платежей | Поддержка российских банковских карт и электронных кошельков, понятный API, быстрая интеграция. |
| **Яндекс.Карты API** | Геокодирование и маршруты | Точные данные для РФ, rich-функционал, умеренная стоимость при небольшом количестве запросов. |
| **Firebase Cloud Messaging** | Push-уведомления | Бесплатно для разумных объёмов, кроссплатформенно, надёжно. |
| **SendGrid (или SMTP)** | Email-рассылки | Простой API, хорошая доставляемость, бесплатный тариф для старта. |
| **S3 (VK Cloud / MinIO)** | Файловое хранилище | Масштабируемость, низкая стоимость, совместимость с большинством библиотек. |

**Принцип выбора:** для MVP главное - скорость разработки и минимальные затраты, поэтому используем знакомые Python/FastAPI, Flutter и уже готовые облачные сервисы для внешних интеграций. На этапе масштабирования стек можно заменить на более производительные компоненты, но в первой версии это не требуется.
